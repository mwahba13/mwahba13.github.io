Lifebrush

VR environment for molecular modelling created in Unreal Engine

Lifebrush is a virtual reality environment created in Unreal Engine which simulates molecular interactions using multi-agent systems. Lifebrush was the PhD project of Dr. Timothy Davison. In the summer of 2020, I was fortunate enough to be given the opportunity to expand on Lifebrush, creating tools and documentation to make it more accessible to a general audience. In particular, I was interested in how we can bridge the gap between programmers and biologists and make Lifebrush user-friendly enough in the hopes that one day it could be used in a classroom. Over the summer months of 2020 i designed and developed a tutorial level for Lifebrush using Unreal Blueprints. The biggest learning curve was figuring out how to take advantage of the VR environment. Designing for VR requires a whole new way of thinking and approaching problems. You can't just use the same solution you would for a non-VR application because then you might as well just make it a non-VR application. I am thrilled to be continuing work on Lifebrush through the Fall 2020 semester by creating documentation and using Lifebrush's C++ API to create a simulation of the Covid-19 virus.

Checkered

Term project for CPSC 585 (Games Programming)

CPSC 585 (Games Programming) is a class offered at the University of Calgary where students are challenged to build a game from the ground up. We were not allowed to use any game engines, only the libraries at our disposal and the skills we learned in previous courses. Checkered is our take on Crazy Taxi. You must survive as an Uber driver trying to make money in a world where Taxi drivers are fed up with Uber and will try to run you off the road. My main contributions to the project was using the PhysX library to set up the physics of our world, setting up the passenger system which involves picking up, dropping off passengers and then receiving a grade based on your drive time, as well as creating some of the UI art assets. This project let me get my hands dirty working on a scrapped together C++ game engine and PhysX. What surprised me was that the most challenging aspect was fine-tuning the driving parameters of the player's car to get it feeling just right.

Ozymandias

Short narrative based on Percy Bysshe Shelley's poem

I met a traveller from an antique land...

So begins Percy Bysshe Shelley's immortal poem about hubris and the tragedy of ego. For one of my first Unity projects, I set out to capture the essence of this poem using nothing but setting. No dialogue, and not much action, Ozymandias is my attempt to evoke a work of art using minimalism. The result is a fantastical tableau which resurrects the glory of a shattered visage among a decaying colossal wreck. This project taught me the basic use of many of Unity's systems including: Trigger Volumes, Audio and Animation. In this project, I also learned how to create a portal effect where it appears as if the player is looking through a doorway into another world. It is a really fun effect as it replicates non-euclidean space, something I plan on doing more experimentation with in the future.

I did not create any of the art assets for this project, only the design and scripting. Art assets were obtained from the Unity Asset store.

The Red Room

"I have no idea where this will lead us, but I have a definite feeling it will be a place both wonderful and strange."

One of my first forays into Unity development involved recreating an iconic environment from one of my favorite shows: Twin Peaks. Since this was one of my first Unity Projects I started dreadfully simple. All the player can do is look around a poorly lit room and take in the music. Although the visuals are not very pleasing, this project taught me a lot about Unity development. First, I learned that EVERYTHING is harder than it looks when it comes to development. Second, I learned the basic Unity workflow. How to set up materials, textures, and audio in a scene. There isn't anything that is scripted or animated in this sample, as that would come a bit later in my Unity development career. It is clear from this sample that there was still much for me to learn at the time, and learn I did.

I did not create any of the art assets for this project. All assets came from the Unity Asset Store.

VR Input Logger

Utility for logging VR device input

This utility was developed after a friend of mine was having a problem. It was necessary for her research to track second-to-second metrics of an HTC Vive's position and controller state. There are programs which do so, however they cost money. I set out to develop a tool which she could use to make her life easier. I wrote this tool in C++ to use the OpenVR library to track the position of both controllers and the head-mounted display, and then output these datapoints over a period of time into a csv file for easy analysis. The interface is a simple command-line interface which allows the user to start and stop the logging of their movement. Not only was this one of my early experiences developing a tool using an existing API, but it was also a great exercise in requirement engineering. The whole process started with a need that need to be fulfilled, and required me to go back and forth to ensure it was a tool that could be put to good use.
