03 · What got better

The wins, in order of impact.

These are the changes a recruiter will feel in the first 30 seconds.

F-01

Hero now leads with proof, not pillars

Fixed

The abstract "Design · Engineering · Narrative" eyebrow is gone, replaced with the credential ledger the audit asked for. A recruiter sees Maxis, USC MFA, AWE Best in Show, and the two shipped Steam titles inside the first viewport. CTAs are de-escalated to one primary ("See the work") plus a single Resume secondary, which is the exact funnel the audit recommended.

What landed

  • Credential ledger: Now · Education · Awards · Shipped as a typed table.
  • Italic accent on the proper nouns (EA Maxis, USC, AWE, Egregore) does the visual work the original 60-word lede was trying to do.
  • "Software Engineer · Game Designer" eyebrow maps cleanly to a job rec.
  • One primary CTA, one secondary, no more six-button hero.

One small tweak

The audit recommended a role line under the H1. Currently the role sits in the eyebrow above it. Both work, but a sub-display line ("Software engineer and game designer." directly under the name, in serif italic) would give the hero a stronger first beat. Low priority, but worth trying — the eyebrow can stay as the credential marquee.

F-02

Single front door for hiring traffic

Fixed

Index and portfolio have collapsed into one document. Featured work, About, and Press all live on index.html. The coaching tease has been commented out of the homepage, sending coaching prospects to their own page via the navbar only when relevant. Nav trims to Work · About · Press · Contact + Resume CTA — exactly the four-link spec.

Notable

  • The press section is now on the hiring page, not buried on a separate homepage. Social proof in the right room.
  • Email is the contact CTA, both in nav and in the about-aside.
  • Coaching CTA card is preserved in source as a commented block so it can come back as a footer mention without retyping.

Open follow-up

The audit also suggested a "now" line in the footer ("Currently at Maxis · open to chats"). Footer is still link-row + copyright only. Adding a one-sentence "now" line above the link row would close the loop with the hero credential.

F-03 / F-08

Editorial system is now site-wide

Fixed

The dark slab + amber template is gone. Every page (index, maxis, egregore, immersive-archive, atrio, coaching) now pulls from css/tokens.css — warm cream #f7f4ee, single rust accent #c4541a, Playfair Display display + Poppins body. Italic-accent emphasis used as the only chrome flourish. This is the most decisive upgrade in the redesign: the site reads senior now.

What's strong

  • One canonical token file. Cards, asides, tags, buttons all share variables.
  • Tags rendered as outlined mono-style labels, not filled pills. Audit ask, delivered.
  • Hover treatment on cards is restrained: lift + border + image scale. No glow, no overlay gradient, no title color flip.
  • Italic accent emphasis on proper nouns is the single best move — it gives every page a consistent voice.

One typographic gap

The audit proposed a three-family system: serif display + sans body + mono for metadata. The current tokens stop at two — Playfair + Poppins — with eyebrows rendered in tracked-uppercase Poppins doing the mono role. It works, but adding JetBrains Mono (or similar) for eyebrows, file labels, and stat numbers would make "this is metadata" visually unmistakable. Optional but cheap.

F-04

Flagship-first hierarchy is clear

Fixed

The featured grid is now exactly the four projects the audit recommended: Maxis, Egregore, Immersive Archive, Atrio. Everything else (Quidditch Swarm, Antymology, L, Limina) has been pulled off the page. The secondary tier is a four-row link list: Virtual Garden, Radio Exurbia, "More on itch.io," "More on GitHub." This is the right shape.

Why it works

  • The four-card grid uses identical 16:9 framing, which gives the flagships consistent visual weight.
  • The secondary list reads as "there's more if you want it", not as competing portfolio entries.
  • External links (itch, GitHub) handle the long tail without bloating the page.

One thumbnail still off

The Maxis card uses the office/studio photo — fine as a case-study hero, but the audit specifically flagged it as a poor portfolio thumbnail. A tool screenshot or in-game capture would land harder in the grid. See F-09 below.

F-05

Case studies now have outcomes

Fixed

Every project page has been rebuilt around a Problem / Approach / Outcome (PAO) block, exactly as the audit requested. The outcomes are concrete: "Best in Show, AWE 2024", "Mostly Positive on Steam", "co-authored IEEE paper", "shipped tools used daily by the Maxis dev team". The aside on each page lists Role, Stack, Recognition, and Artifacts as scannable rows.

Standouts

  • Immersive Archive — outcome line now reads as a literal CV bullet: Best in Show + SIGGRAPH + IEEE in one sentence.
  • Egregore — Pitch Deck and Design Doc are promoted into the aside as artifact links. That's the audit ask.
  • Maxis — finally describes Sims Maker Suite and Creator Program as a concrete shipped surface.

Still missing one artifact per page

The audit asked for one artifact beyond a video — a code snippet, a system diagram, a Blueprint screenshot, a design-doc page. The case studies have trailers and key art but no showing of the work itself. The Egregore graph-system is described in two paragraphs that beg for a diagram. Maxis is asking for a tool screenshot. Atrio could use a Twine-prototype capture. One concrete artifact per case study is the next 80/20 lift.

05 · New since v1

Things the redesign introduced.

Small, but they're real. Catch them now before they bake in.

NEW-01

Em-dashes throughout the case studies — your own style guide forbids them

New Minor

CLAUDE.md is explicit: "No em-dashes. Use commas, periods, or restructure the sentence instead." The new case studies are full of them. A quick scan:

Examples

  • immersive-archive.html — "The challenge wasn't just technical — the originals no longer run"; "Preservation isn't nostalgia — it's the difference…"; "technical roadmap for a 10-person team, working under XR pioneer Scott Fisher — how recreations should be structured…"
  • atrio.html — "a setting where light is currency and the dark is something you negotiate with" (clean); but "Build lore that hooks into the survival loop — light as scarce, hope as scarce — and a dialogue system…"; "That cycle — write, review, test, revise — drove most of the final shape…"
  • index.html hero credentials — "Software Engineer at EA Maxis — The Sims tools", "MFA, Interactive Media — USC", etc. (Could be replaced with a comma or restructured row.)

Fix

Find/replace — across *.html. Most cases are happy as a period, comma, or restructured clause; a few credential lines read more cleanly with · or simply a comma. Worth doing in one batched pass before the next external share.

Set aside file titles and OG meta titles ("Egregore — Michael Wahba") — those use em-dashes by convention and don't need to change.

NEW-02

Coaching tease is commented-out, not deleted

New Minor

The coaching CTA card on the homepage is wrapped in <!-- … -->. That's a fine staging state, but if it's not coming back, delete it. If it is coming back as a footer mention (which the audit recommended), commit to that placement now so the source is honest about intent.

NEW-03

README is out of date with the actual repo

New Minor

README.md claims portfolio.html and modals.html are "superseded; not present in this build." Both are still on disk. Either delete the files and the claim becomes true, or update the README to reflect what shipped. (Worth doing alongside the F-10 cleanup so the next reader trusts the README.)

NEW-04

Hero credential ledger is long

New Minor

Nine credential rows in the hero (Now / Education ×2 / Awards ×4 / Shipped ×2) is more than the audit anticipated. It's not wrong, but on a tall viewport it competes with the portrait and pushes the four featured cards below the second fold. Worth considering:

Options

  • Keep only the Now / Education (USC) / Award (AWE) / Shipped (Egregore + Atrio) top 5. Move the rest to a dedicated "Honors & recognition" strip lower on the page.
  • Or: group the four scholar/award rows under a single Honors label with the names inline, comma-separated. Fewer rows, same density of proof.
  • Or: keep all nine but tighten padding on .credential-row from 0.95rem 0 to 0.7rem 0 so the block doesn't push the work below the fold.

Why it matters

The audit's whole F-01 thesis is that the proof should appear above the fold. Right now the proof is there, but it's everything in the first viewport. The work itself should still be visible (even partially) on a 1080p desktop. Currently it's not.

NEW-05

Footer is bare

New Minor

It has the four social links and a copyright line. The audit's IA recommendation was to add: contact, a "now" line, and a small sitemap. The "now" line in particular ("Currently at EA Maxis · Open to chats.") bookends the hero's credential ledger nicely and gives a hiring manager the takeaway sentence to remember.

06 · Roadmap

If you had four hours this weekend, do these in order.

Sequenced so each step keeps the site shippable. None of these require new design work — they're all execution.

  1. Delete stale files. Remove modals.html, the duplicate markup in portfolio.html (leave the meta-refresh), and docs/Curriculum Vitae, Wahba (May 2025).pdf. Update README.md to match.
    ~20 min
  2. Em-dash sweep. Find/replace — across HTML bodies (skip <title> and OG metas). Most become periods, commas, or middle-dot separators. Reread the affected paragraphs once for flow.
    ~30 min
  3. Repo hygiene sweep (F-10). Delete the dead CSS / JS / SCSS / fonts directories enumerated in §04. Keep only tokens.css, load-navbar.js, and any icon set still referenced. Verify each page in a browser; commit. This is the single biggest credibility win in the codebase.
    ~45 min
  4. Add one artifact per case study. One concrete showing: a screenshot, a Blueprint capture, a system diagram, a Twine map, a code snippet styled in mono. Egregore's graph mechanic and Atrio's Twine prototyping are the strongest candidates.
    ~90 min
  5. Trim hero credentials, add footer "now" line. Drop from nine rows to five or six (per NEW-04). Add "Currently at EA Maxis · Open to chats." above the footer link row. Bookends the page.
    ~20 min
  6. Accessibility close-outs. Skip-to-content link at the top of <body>. Promote the Press .section-label paragraph to an <h2>. Five minutes each.
    ~15 min
  7. Better Maxis card thumbnail. Replace the office photo with a tool screenshot, an in-game capture from a shipped Sims surface, or a clean typeset card. Same crop, same ratio, so the four flagships keep their visual unity.
    ~30 min
  8. OG card image + favicon (optional polish). Generate a 1200×630 OG card for the homepage with the credential line baked in. Replace stockController.png favicon with a custom monogram. Add JSON-LD Person schema to the homepage <head>.
    ~45 min

Total: roughly four hours. After that, the site is finished against the original audit and ready to share with no asterisks.

In closing

The hard part is done.

The v1 audit was, at its core, a critique of how the site was selling the work. The redesign answered that. The credential ledger, the editorial system, the four-flagship grid, the PAO blocks — these are the moves that turn a personal site into a hiring portfolio.

What's left is mostly subtraction, not addition. Delete the dead files. Strip the dead deps. Add one artifact per case study, one "now" line in the footer, and a few accessibility close-outs. The portfolio is a strong working draft today. After a half-day pass it's done.
